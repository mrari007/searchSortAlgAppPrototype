Learn Searching and Sorting Algorithms with Java


In computer science, an algorithm is a sequence of well-defined instructions, usually used to solve a specific problem or to perform a computation. As the names suggest: searching algorithms' function is to retrieve information stored within some sort of data structure (array, Arraylist, Linkedlist, etc.). On the other hand, the task of sorting algorithms is to put the items of a list into an order.

Big O Notation


The most important thing about algorithms is their performance which is measured in Big O notation. Big O notation is a mathematical notation that describes the limiting behavior of a function when the argument tends towards a particular value or infinity. In computer science, big O notation is used to classify algorithms according to how their run time or space requirements grow as the input size grows. (Visit wikipedia for more information) There is no universal algorithm that can be used in any possible scenario and work better that any other. Different algorithms perform differently depending on the specific situation and the size of the data structure they will be applied to.

Time complexity

Time complexity describes the amount of time it takes to run an algorithm. The most commonly used method to estimate a performance of an algorithm is to consider its worst-case complexity. In the case of running time, the worst-case time-complexity indicates the longest running time performed by an algorithm given any input of size n. For example, the worst time complexities of sequential and binary search are O(n) and O(log n) respectively

To visualize it more clearly imagine that you have to search for an element in an array with length n and you will be using Sequential search. Sequential search checks each element of the array until a match is found or the entire array has been searched. Therefore, the worst time complexity is if the searched element is at the last position of the array or is not present at all. On the other hand, the best time complexity is if the element is at position O. That means that in the best time complexity of sequential search is O(1) and the worst is O(n)

Space complexity

Space complexity similarly to time complexity is measured with Big O. The difference is that while time complexity is concerned with the run time of the algorithm, space complexity is all about the amount of memory space that is required by the algorithm until it executes completely. Unfortunately space and time are two separate poles. Incrementing speed will most often lead to memory consumption growth and vice-versa. For example merge sort is exceptionally fast but takes a lot of memory. On the contrary bubble sort is relatively slow but requires less memory.
